#### PR-4.2(2026-05-01):check_open_prs 邏輯瑕疵 — 留 follow-up

**症狀:** check_open_prs 用 `git merge-base --is-ancestor <pr_head_sha> origin/main` 判斷 PR 是否 merged。對 squash / rebase merge 的 PR 誤判:
- GitHub `refs/pull/N/head` 存的是 PR HEAD commit(merge 前 branch tip)
- squash / rebase merge 後,merge commit ≠ PR HEAD commit
- PR HEAD commit 不在 main 的 reachable 範圍 → ancestor check 回 false → 誤判 open

**真打觀察:** 2026-05-01 主 server 跑 verify_state.py 列「12 open PRs: [2, 16, 17, 18, 19, 20, 21, 22, 23, 24, 79, 80]」,但其中至少 #79(PR-3.10 後 cleanup)是 merged 狀態。
